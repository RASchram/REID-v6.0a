# REID v6.0a — CHANGELOG

This changelog documents the public-safe evolution of the REID architecture leading into the v6.0a release. It reflects structural, architectural, and governance-level changes only. Identity-locked, substrate-level, and private build-sequence details are intentionally excluded.


## Version 6.0a (June 2026)

### Overview
REID v6.0a introduces a consolidated four-layer architecture spine, expanded invariants, refined requirements governance, and an enterprise-ready integration surface. This release represents the next major evolution of the Relational Emergent Identity Dynamics framework.

### Additions
- Added Four-Layer Spine Overview (A-09)
- Added Requirements Architecture (A-12)
- Added Enterprise Integration Overview (A-13)
- Added Cross-Layer Binding Rules (Overview)
- Added public-safe lineage documentation derived from v5.0a
- Added public-safe dependency schema (DGS-0002-v2.0)
- Added public-safe traceability matrix standard (R-20)

### Updates
- Updated architectural terminology for consistency across layers
- Updated definitions and invariants to align with v6.0a governance
- Updated requirements lifecycle governance for cross-layer compatibility
- Updated dependency graph notes for clarity and public-safe release

### Removals (Intentional Exclusions)
The following components remain private and are not included in the public-safe v6.0a repository:
- Meta-Substrate mechanics
- Mode engines
- Closure and Immunity Systems
- Orchestration Spine internals
- Build sequencing and manifest logic
- Identity-Lock parameters
- All private artifacts from the 46-item authoritative build suite


## Version 5.0a (June 2026)

### Overview
REID v5.0a established the stable alpha baseline for core definitions, invariants, and lineage continuity. These materials form the foundation for the public-safe portions of v6.0a.

### Additions
- Core Definitions and Invariants (v5.0a)
- Version Lineage and Continuity Record
- Integration Summary (public-safe portions)

### Notes
v5.0a artifacts are included in the v6.0a repository only where they support public architectural understanding. Identity-locked content remains private.


## Version 3.1.1a (June 2026)

### Overview
REID v3.1.1a served as the unified specification corpus and the first stable alpha baseline for the REID governance framework.

### Status
Deprecated. Preserved as a historical baseline. Superseded by REID v6.0a.

### Notes
No further updates will be made to v3.1.1a. The repository remains archived for lineage continuity.


## Version History Summary

- v6.0a: Canonical architecture release; unified spine; expanded invariants; enterprise integration surface.
- v5.0a: Stable alpha baseline for definitions, invariants, and lineage.
- v3.1.1a: Unified specification corpus; deprecated and archived.
